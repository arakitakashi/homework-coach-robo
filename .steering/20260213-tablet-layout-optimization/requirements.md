# Requirements - タブレット横長レイアウト最適化

## 背景・目的

現在のセッション画面は縦長レイアウトで設計されており、タブレット端末（横長画面）で使用する際に以下の問題がある：
- 全てのコンポーネントが縦に並んでいるため、スクロールが必要
- タブレットの横長画面の利点を活かせていない
- 小学校低学年の児童が使用する際、重要な情報（キャラクター・対話履歴）が画面外になることがある

タブレット使用を見越して、横長画面（1024x768）でスクロールなしで全体が一覧できるレイアウトに改善する。

## 要求事項

### 機能要件

1. **レイアウト構成**
   - タブレット横長画面（1024x768）でスクロールなしで全体表示
   - 重要な要素（キャラクター・対話履歴）を大きく表示
   - サブ情報（進捗・感情）をコンパクトに配置
   - 既存のコンポーネントをすべて配置（機能削減なし）

2. **レスポンシブ対応**
   - タブレット横長（1024x768以上）: 2カラムレイアウト
   - タブレット縦長・スマートフォン（768px未満）: 既存の縦長レイアウト
   - デスクトップ（1280px以上）: 2カラムレイアウト（より広い）

3. **ユーザビリティ**
   - タッチ操作に適したサイズ（ボタン最小44x44px）
   - 視認性の高い配色
   - 直感的な情報配置

### 非機能要件

1. **パフォーマンス**
   - レイアウト変更による既存機能への影響なし
   - レンダリングパフォーマンスの維持

2. **保守性**
   - 既存コンポーネントの再利用
   - CSS GridまたはFlexboxによる保守しやすいレイアウト
   - Tailwind CSSを使用した一貫性のあるスタイリング

3. **アクセシビリティ**
   - セマンティックHTML
   - ARIA属性の適切な使用
   - キーボードナビゲーション対応

### 制約条件

1. **技術的制約**
   - Next.js 16 App Router
   - Tailwind CSS
   - Framer Motion（既存アニメーション維持）
   - Jotai（状態管理）

2. **デザイン制約**
   - 小学校低学年向けの親しみやすいデザイン
   - 既存のキャラクター・カラースキームとの整合性
   - 明るく楽しい雰囲気の維持

## 対象範囲

### In Scope

- SessionContentコンポーネントのレイアウト変更
- タブレット横長画面でのレスポンシブ対応
- CSS GridまたはFlexboxによる2カラムレイアウト実装
- 既存コンポーネントの配置最適化
- テストの更新（レイアウト変更に伴う）

### Out of Scope

- 個別コンポーネントの内部ロジック変更
- 新機能の追加
- バックエンドAPI変更
- スマートフォン縦長レイアウトの大幅変更（既存維持）

## 成功基準

1. **視覚的完成度**
   - タブレット横長画面（1024x768）でスクロールなしで全体表示
   - 重要な要素が適切に強調されている
   - 小学校低学年にとって見やすく、使いやすい

2. **機能的完成度**
   - 既存の全機能が正常動作
   - レスポンシブ対応が正しく動作（768px未満は縦長、1024px以上は横長）
   - テストが全て通過

3. **品質基準**
   - リンターエラーなし（Biome）
   - 型チェックエラーなし（TypeScript）
   - テストカバレッジ80%以上維持
